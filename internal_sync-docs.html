TestConcurrentCache

TestConcurrentCache tests HashTrieMap in a scenario where it is used as the basis of a memory-efficient concurrent cache. We're specifically looking to make sure that CompareAndSwap and CompareAndDelete are atomic with respect to one another. When competing for the same key-value pair, they must not both succeed. This test is a regression test for issue #70970.
